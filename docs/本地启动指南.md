# shopkeeper-agent 本地启动指南

> 本文档记录本机（Windows）下启动该项目的**完整步骤、踩过的坑和专属配置**。
> 以后启动项目，照这一份文档操作即可。

---

## 〇、一句话总览

这个项目要同时跑起来 **3 样东西**，缺一不可，且有先后顺序：

```
① Docker 五个容器（数据库 / 检索 / Embedding）
        ↓ 必须先就绪
② 后端  FastAPI（端口 8000）
        ↓
③ 前端  Vite（端口 5173）
```

启动后访问 **http://localhost:5173** 使用。

---

## 一、本机专属配置（重要，和项目默认不一样）

为避开本机端口占用，**改过 3 个端口**，已写入配置文件，无需再改：

| 服务 | 项目默认端口 | 本机改成 | 原因 |
| --- | --- | --- | --- |
| MySQL | 3306 | **3307** | 避开本机已有的 MySQL |
| Elasticsearch | 9200 | **19200** | 9200 被 cpolar 占用 |
| Kibana | 5601 | **15601** | 5601 在 Windows 保留端口段 |
| Qdrant | 6333 | 6333（未改） | — |
| Embedding | 8081 | 8081（未改） | — |

> 改动落在两个文件：
> - `docker/docker-compose.yaml`（容器端口映射）
> - `conf/app_config.yaml`（后端连接配置，`es.port=19200`、两个 MySQL `port=3307`）

## 二、方式 A：一键脚本启动（推荐日常使用）

项目根目录已准备好两个脚本：

| 脚本 | 作用 |
| --- | --- |
| `start.bat` | 一键启动 Docker + 后端 + 前端，最后自动打开浏览器 |
| `stop.bat` | 停止 Docker 容器 |

### 启动步骤

1. **先打开 Docker Desktop**，等托盘鲸鱼图标不再转动（变绿）。
2. 用 **Windows 资源管理器**（不是 VS Code 文件树）进入项目目录 `E:\app\agent\shopkeeper-agent`。
3. **双击 `start.bat`**。
4. 会弹出 3 个黑窗口：
   - 主窗口：显示进度
   - `shopkeeper-backend`：后端日志，出现 `Application startup complete` 即就绪
   - `shopkeeper-frontend`：前端日志，出现 `Local: http://localhost:5173/` 即就绪
5. 等后端就绪后，回主窗口**按任意键**，浏览器自动打开 http://localhost:5173。

> ⚠️ `shopkeeper-backend` 和 `shopkeeper-frontend` 两个窗口**使用期间不要关闭**（可最小化）。关闭即停止服务。

### 停止步骤

1. 关掉 `shopkeeper-backend`、`shopkeeper-frontend` 两个黑窗口。
2. 双击 `stop.bat` 停止 Docker 容器。

### 在终端里运行脚本（注意）

本机设置了 `NoDefaultCurrentDirectoryInExePath`，在终端里**不能直接输 `start.bat`**，必须带 `.\` 前缀：

```bash
.\start.bat
.\stop.bat
```

（资源管理器里双击不受此影响，走的是完整路径。）

## 三、方式 B：手动命令启动（理解原理 / 排查问题时用）

在项目根目录 `E:\app\agent\shopkeeper-agent` 下，按顺序操作。

### ① 启动 Docker 容器

```bash
docker compose -f docker/docker-compose.yaml up -d
```

确认五个容器都 `Up`：

```bash
docker ps
```

应看到：`mysql`、`elasticsearch`、`qdrant`、`embedding`、`kibana`。

> 容器配了 `restart: unless-stopped`，Docker Desktop 启动后通常会自动拉起，这一步多为「确认」。

### ② 启动后端（必须带 UTF-8 前缀）

开一个终端：

```bash
PYTHONUTF8=1 PYTHONIOENCODING=utf-8 uv run fastapi dev main.py
```

看到 `Application startup complete` 即成功。后端地址：http://127.0.0.1:8000

> ⚠️ **不能直接 `uv run fastapi dev main.py`**！否则启动时输出 emoji 会让 Windows GBK 控制台崩溃（报 `UnicodeEncodeError`）。必须加 `PYTHONUTF8=1 PYTHONIOENCODING=utf-8`。

### ③ 启动前端

**另开一个**终端：

```bash
cd frontend
pnpm dev
```

看到 `Local: http://localhost:5173/` 即成功。浏览器访问该地址即可。

## 四、首次 / 重置后才需要的一次性步骤

以下步骤**平时启动不需要**，只有第一次部署、或删除了 Docker 数据卷后才需要。

| 步骤 | 命令 | 说明 |
| --- | --- | --- |
| 安装后端依赖 | `uv sync` | 安装 Python 依赖到 `.venv` |
| 安装前端依赖 | `cd frontend && pnpm install` | 安装前端 node 依赖 |
| 构建元数据知识库 | `uv run python -m app.scripts.build_meta_knowledge -c conf/meta_config.yaml` | 把元数据写入 MySQL / Qdrant / ES |

> 改了 `conf/meta_config.yaml`（新增表 / 字段 / 指标）后，需重新执行「构建元数据知识库」。

## 五、验证是否正常（冒烟测试）

后端起来后，可不经前端直接测接口（用 Python 发，避免 Windows 终端中文乱码）：

```bash
PYTHONUTF8=1 .venv/Scripts/python.exe -c "import httpx; \
[print(l) for l in httpx.stream('POST','http://127.0.0.1:8000/api/query', \
json={'query':'统计华北地区的销售总额'}, timeout=120).__enter__().iter_lines() if 'result' in l]"
```

预期返回类似：`data: {"type": "result", "data": [{"销售总额": 41099.5}]}`

或直接在网页 http://localhost:5173 输入问题测试。

## 六、常见问题（FAQ）

### Q1：网页查询报 `502 Bad Gateway from localhost:8081`
**原因**：Embedding 容器（8081，负责把文字转向量）冷启动需要约 **4~5 分钟**加载模型，期间未就绪。
**解决**：等几分钟，浏览器打开 http://localhost:8081/health 不报错后，再去网页提问。
> Docker 一直开着的话，热身只在容器**刚启动那次**发生，之后一直是热的。

### Q2：双击 `start.bat` 没反应
**原因**：在 **VS Code 文件树里双击**只是打开文件，不会运行。
**解决**：用 **Windows 资源管理器**双击，或终端里 `.\start.bat`。

### Q3：终端输 `start.bat` 报「不是内部或外部命令」
**原因**：本机设了 `NoDefaultCurrentDirectoryInExePath`，禁止从当前目录找程序。
**解决**：加前缀 `.\start.bat`。

### Q4：后端窗口报 `invalid PYTHONUTF8 environment variable value`
**原因**：环境变量值带了多余空格（如 `PYTHONUTF8=1 ` 末尾空格）。
**解决**：确认设置为干净的 `PYTHONUTF8=1`（脚本已修复）。

### Q5：前端提示 `Port 5173 is in use, trying 5174`
**原因**：5173 已被另一个前端进程占用。
**解决**：关掉占用 5173 的旧进程，或直接用它退而求其次的 5174 端口访问。
